Unlambda Reference

All data in Unlambda is made up of functions. Each functions in Unlambda takes one parameter and returns a function.

Unlambda has one operator, the apply operator. It is represented with ` (a grave). This operator applies a function to another function. It is used as such: `AB applies the function A to the function B.

For the purposes of this module, only two base functions will appear: s and k. All other Unlambda functions will never appear on this module.

The k function, when applied to a function A, returns a function, that when applied to a function B, returns A. In other words, k creates a constant function that ignores its input and returns A.

The s function is more complicated than the k function. When applied to a function A, it returns another constructor, which returns a third function when applied to a function B. This third function, when applied to a function C, returns the result of ``AC`BC. In other words, the previous expression is equivalent to ```SABC.

In Unlambda, an expression `AB always gets the result of A, then gets the result of B, then applies the one to the other.
